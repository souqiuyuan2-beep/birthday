# 誕生日サプライズWebサイト

恋人の誕生日旅行を「体験型サプライズ」にするWebアプリ。QRコード→合言葉→手紙のオープニング→スポットごとのミッション(写真撮影)→エンディングのスライドショー→思い出アルバム、という流れ。

## 必読ドキュメント
- `docs/birthdayWeb.md` — 要件定義書 v1.1(機能・非機能要件、デザイン方針)
- `docs/開発計画.md` — 開発計画書(DB設計、画面一覧、実装順序、MVP定義)

## 技術スタック
Next.js 15 (App Router) / TypeScript / Tailwind CSS v4 / Framer Motion / Supabase (Postgres + Storage + Realtime) / Resend(メール通知)/ Vercel

## 重要な前提
- メインユーザーは iPhone Safari(375〜430px基準、モバイルファースト)
- 弱電波前提: 画像はクライアント圧縮(長辺1600px/約500KB)+リトライ付きアップロード
- BGMはユーザーのタップを起点に再生(iOS Safari自動再生制限)
- 写真・BGMは非公開バケット+署名付きURL。EXIF位置情報は除去
- 合言葉(彼女用)と管理者パスワード(自分用)は別系統。合言葉・書き込みの検証はサーバー側で行う
- 通知はメール(Resend)。要件書内のDiscord記述は旧案(§8-7で上書き済み)
- 無料枠で運用(Vercel Hobby + Supabase Free + Resend Free)

## 実装順序(docs/開発計画.md §5)
1. 雛形+DB(シードでホーム表示)→ 2. コア体験(ミッション→撮影→達成)→ 3. 簡易アルバム【ここまでMVP】→ 4. 管理画面 → 5. ダッシュボード+通知 → 6. オープニング → 7. エンディング → 8. 磨き込み → 9. デプロイ

## 規約

## 1. ディレクトリ構成

Next.js(App Router)+ TypeScript。旅行ごとにランダムなslug(推測されにくいURL)を採用。

```

│   ├── layout.tsx                  # ルートレイアウト(フォント・メタ)
│   ├── globals.css
```

## 2. 使用技術

| レイヤー | 技術 | 補足 |
|---|---|---|
| フレームワーク | Next.js 15(App Router)+ React + TypeScript | Vercelデプロイ前提 |
| スタイリング | Tailwind CSS v4 | モバイルファースト(375〜430px基準) |
| アニメーション | Framer Motion | 手紙・スタンプ・達成演出・ページめくり |
| DB / Storage / Realtime | Supabase(Free) | プラン・進捗・写真・BGM。写真/BGMは非公開バケット+署名付きURL |
| メール通知 | Supabase Database Webhook → Edge Function → Resend(無料枠) | photos INSERT時に管理者へメール |
| 画像圧縮 | browser-image-compression | 長辺1600px / 約500KB。EXIF位置情報除去 |
| ホスティング | Vercel(Hobby) | 無料・HTTPS |
| BGM再生 | HTML Audio + ユーザータップ起点で再生 | iOS Safari自動再生制限対策 |
| QRコード | qrcodeライブラリで生成(開発とは独立) | 印刷用 |

## 3. データベース設計(Supabase / Postgres)

### trips(旅行)
| カラム | 型 | 備考 |
|---|---|---|
| id | uuid PK | |
| slug | text UNIQUE | ランダムURL用(例: 20文字英数) |
| title | text | 表紙タイトル |
| date | date | 旅行日 |
| passphrase | text | 合言葉(記念日) |
| opening_letter | text | オープニングの手紙 |
| ending_letter | text | エンディングの手紙 |
| opening_bgm_path | text NULL | Storageパス(管理画面からアップ) |
| ending_bgm_path | text NULL | 同上(未設定ならオープニング曲を流用) |
| notify_email | text NULL | 通知先メールアドレス |
| theme_color | text | 既定 '#A8D8EA'(薄い水色) |
| created_at | timestamptz | |

### spots(スポット / ミッション)
| カラム | 型 | 備考 |
|---|---|---|
| id | uuid PK | |
| trip_id | uuid FK→trips | |
| sort_order | int | 並び替え用 |
| name | text | スポット名 |
| reveal_name | bool | 事前に名前を見せるか |
| mission | text | ミッション本文 |
| hint | text NULL | 折りたたみ表示 |
| message | text NULL | 一言メッセージ |

### progress(達成状況)
| カラム | 型 | 備考 |
|---|---|---|
| trip_id | uuid FK | 複合PK(trip_id, spot_id) |
| spot_id | uuid FK | |
| completed_at | timestamptz | 1枚目アップロード時に記録 |

### photos(写真)
| カラム | 型 | 備考 |
|---|---|---|
| id | uuid PK | |
| trip_id / spot_id | uuid FK | |
| storage_path | text | `photos/{trip_id}/{spot_id}/{timestamp}.jpg` |
| created_at | timestamptz | INSERTでWebhook→メール通知 |

### Storage(非公開バケット)
- `photos/` : 上記パス。署名付きURLで表示
- `bgm/{trip_id}/...` : 管理画面からアップロードしたmp3

### アクセス制御
- 匿名キーでのアクセスは最小限にし、読み書きはRoute Handler(サーバー側)経由+RLSで防御
- 彼女側: slug + 合言葉検証済みセッションのみ progress/photos に書き込み可
- 管理側: 管理者パスワード検証済みのみ trips/spots を編集可

## 5. 実装順序

要件書§7のフェーズを踏襲しつつ、「早く通しで動くもの」を優先する順に調整。

1. **雛形+DB**: Next.js雛形、Supabaseプロジェクト、テーブル+RLS、シードデータ投入 → DBのプランがホーム画面に出る
2. **コア体験**: 合言葉(簡易版)→ホーム→ミッション→圧縮アップロード→達成→次解放の一連フロー
3. **簡易アルバム**: 写真グリッド表示(署名付きURL)→ ここまでで**MVP完成**
4. **管理画面**: ログイン、旅行/スポットCRUD・並び替え、BGM/通知先設定
5. **ダッシュボード+通知**: Realtime進捗・写真フィード、photos INSERT→メール通知
6. **オープニング演出**: 封筒アニメ、手紙表示、BGM再生、スキップ
7. **エンディング+アルバム強化**: スライドショー、最後の手紙、ページめくりモード
8. **磨き込み**: 達成演出、フォント、遅延読み込み、弱電波テスト、iPhone実機2台テスト
9. **リリース**: Vercelデプロイ、QRコード生成、通しリハーサル

## 6. MVP(最低限動く版)

**= 上記ステップ1〜3。当日の体験が最低限成立するライン。**

含むもの:
- 合言葉入力(演出なしのシンプル版)
- ホーム画面(スタンプラリーの簡易版: 達成/挑戦中/ロックの3状態)
- ミッション表示 → カメラ起動 → 圧縮 → アップロード → 達成 → 次解放
- 進捗のDB保存(リロードしても消えない)
- 写真グリッドのアルバム
- プランはシードSQLで投入(管理画面はまだ無し)

含まないもの: 管理画面、Realtime、通知、オープニング/エンディング演出、BGM、リトライの作り込み
